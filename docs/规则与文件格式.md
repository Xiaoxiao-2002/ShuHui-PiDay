# 规则与文件格式

## 共同规则

玩家在棋盘边上选择若干边，最终必须形成一条非空、连通、无分叉的闭环。每个图顶点的已选边数只能是 0 或 2，并且所有已选边必须属于同一个连通分量。

数字提示表示该单元格边界中已选边的数量。没有提示的单元格不施加数量约束。

## 经典数回

棋盘由 `rows × cols` 个方格组成，每格四条边，数字范围为 0–4。

## 特色数回

棋盘用 `row_lengths` 描述每行圆数。数组不能为空、每项为正整数、相邻项必须恰好相差 1。短行的圆心自动位于相邻长行两圆之间。

每个圆是一个六边单元格，边为六段六分之一圆弧；每组三个两两相切的圆还形成一个三边圆隙单元格。相切位置合并为同一图顶点，因此闭环可以在相切点处转向，但仍须满足度数 0 或 2。

π 是数字之外的提示类型。答案曲线中，所有至少与一个 π 单元格相邻的已选圆弧，必须在六种方向上各有且仅有一条。不存在 π 提示时不启用该规则。

## JSON v1

```json
{
  "schema_version": 1,
  "variant": "circle_pack",
  "board": {"row_lengths": [3, 4, 3]},
  "clues": [
    {"cell_id": "circle:0:0", "kind": "number", "value": 2, "locked": false},
    {"cell_id": "circle:1:1", "kind": "pi", "locked": true}
  ],
  "target_solution_edges": ["arc:0:0:0"],
  "analysis": {
    "status": null,
    "solution_edges": null,
    "puzzle_hash": null,
    "elapsed_seconds": null,
    "message": null
  },
  "metadata": {}
}
```

`analysis.status` 为 `null`、`invalid`、`unique`、`multiple` 或 `unsatisfiable`。只有 `unique` 时才保存 `analysis.solution_edges`。题面变化后，内容哈希不再匹配，旧分析会在加载时自动清除。

稳定边 ID：经典横边为 `h:行:列`、竖边为 `v:行:列`；特色弧为 `arc:圆行:圆列:方向`，方向为 0–5。圆 ID 为 `circle:行:列`，三角形 ID 由其三个圆 ID 确定。

## 四级难度

- 初级（Beginner）：合并原简单与中等题，以局部数字、顶点和基础 π 组合为主。
- 中级（Medium）：对应原困难题；目标环覆盖较广，跨圆转移多，提示更稀疏。
- 高级（Difficult）：棋盘长宽约为中级的两倍，局部推理强度相近，但需要维持更长的推理链。
- 专家（Expert）：使用大型棋盘，同时要求长环、高跨圆转移、高覆盖和复杂提示组合，不仅以尺寸制造耗时。

自动筛题仍在 `metadata.difficulty.level` 中保留 1–5 的内部评分分段；玩家可见的四级标签记录在 `metadata.difficulty_tier`，显示名称记录在 `metadata.curated_difficulty`。最终分级仍应由试玩确认。
