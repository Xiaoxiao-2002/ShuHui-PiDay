# πDay 特色数回网页活动版

网页活动版位于 `web/`，是独立于 PySide6 出题工具的 React + TypeScript PWA。参与者无需登录或安装，使用浏览器即可完成 20 道题；首次完整打开后，全部关卡可以离线使用。

## 互动教程

首页的“开始互动教程”提供八步、不计时的教学流程：画线与删除、数字提示、顶点分叉、单一闭环、π 统计范围、六类圆弧、π 完整约束和综合练习。教程允许学生实际制造错误并看到针对性反馈；正式关卡仍只报告错误类别，不泄露具体错误位置。

教程进度保存在当前浏览器中，刷新后会恢复课次，完成后可以从首页重新学习。教程使用独立的演示数据，不读取或展示 20 道正式题的答案；全部内容随 PWA 一起离线缓存。

## 本地开发

要求 Node.js 24 LTS 和本仓库的 Python 3.12 虚拟环境。

```powershell
cd web
npm.cmd ci
cd ..
.\.venv\Scripts\python.exe scripts\export_web_puzzles.py
cd web
npm.cmd run dev
```

生产检查：

```powershell
cd web
npm.cmd test
npm.cmd run verify:puzzles
npm.cmd run test:e2e
npm.cmd run build
```

`verify:puzzles` 会用 TypeScript 验证器复验 20 道正式答案，并递归检查网页题库不含目标曲线、分析答案等字段。Python 导出器还会检查题号连续、难度分布为 8/4/4/4，以及每道题仍为唯一解。

## GitHub Pages 发布

`.github/workflows/deploy-pages.yml` 会在 `main` 更新后重新导出安全题库、运行 Python/TypeScript/Chromium 检查、构建 PWA，并发布 `web/dist`。

首次发布前，在 GitHub 仓库的 **Settings → Pages → Build and deployment → Source** 中选择 **GitHub Actions**。成功后入口为：

`https://xiaoxiao-2002.github.io/ShuHui-PiDay/`

公网二维码位于 `web/public/qr-entry.svg` 和 `web/public/qr-entry.png`。修改公网地址后运行：

```powershell
cd web
npm.cmd run qr -- "https://新的地址/"
```

## 现场局域网镜像

活动电脑与参与者手机连接同一 Wi-Fi，然后运行：

```powershell
.\scripts\start_web_lan.ps1
```

脚本会重建题库和网页、监听 8080 端口，并为每个本机局域网 IPv4 地址显示二维码。若 Windows 防火墙询问，请只允许专用网络。可用 `-Port 9000` 更换端口，或在已经构建后用 `-SkipBuild` 直接启动。

局域网 HTTP 版可在线游玩，但浏览器的“安装到桌面”和可靠离线缓存需要 HTTPS 公网版。活动前应分别用现场 Wi-Fi 和三家移动网络测试公网入口，并保留局域网二维码作为备用。

## 数据与隐私

- 网页发布数据不包含答案；答案仅在构建和测试阶段读取。
- 作答、开始时间和最佳成绩只存于当前浏览器的 `localStorage`。
- 凭证由 `LocalReceiptIssuer` 在本地生成，不上传姓名、手机号或设备标识。
- 截图凭证可被转发或修改，这是首版人工查验方案的已知边界。
- `RemoteReceiptIssuer` 已定义服务端提交格式，未来可接入签名二维码与一次性核销，无需改动游戏流程。

## 缓存更新

新版本发布后，已打开页面会提示刷新。每道题保存时记录数据版本和题目哈希；哈希变化会丢弃该题不兼容的旧作答，已完成记录仍保留其数据版本。
