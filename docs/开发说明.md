# 开发说明

## 架构

- `model.py` / `storage.py`：版本化数据对象、内容哈希与原子保存；
- `topology.py`：经典和圆密堆积棋盘到统一图结构的映射；
- `solver.py`：OR-Tools CP-SAT 精确模型。边为布尔变量，`AddCircuit` 全局约束配合未使用顶点自环，直接保证非空单一闭环；求解默认使用最多 8 个 CPU 工作线程；
- `generator.py`：π 组合枚举和反例驱动提示最小化；
- `difficulty.py`：确定性数字、顶点和 π 推理统计；
- `gui.py`：PySide6 桌面应用和后台工作线程；
- `render.py` / `pdf_export.py`：SVG 与 A4 PDF 输出；
- `pack.py`：随机简单闭环采样。

## 开发命令

```powershell
.\.venv\Scripts\python.exe -m pytest
.\.venv\Scripts\python.exe scripts\build_activity_pack.py --count 60 --seconds 6
powershell -ExecutionPolicy Bypass -File scripts\build_windows.ps1
```

生成器先验证全提示题面并保留唯一解基线，再从基线逐个删除提示、每步精确复验；之后由主问题选择数字提示集合，并要求每个已知替代解至少被一个提示排除。子问题在当前提示下寻找不同于目标曲线的新解。到达时限只返回已经完成唯一性证明的候选，不把“暂时没找到第二解”当作唯一。

难度标签综合确定性传播剩余比例、目标环长度、棋盘覆盖范围和跨圆转移次数，再由人工试玩覆盖。短小单元格周界不会再因提示少被误标为困难。自动分数仍是筛选工具，不是人类完成时间的替代品。
